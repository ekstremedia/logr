## Phase 3: Multi-Window Management - In Progress

### Rust Backend - Window Commands
- Created `window_commands.rs` with Tauri commands:
  - `create_log_window` - Create a new window for a log source
  - `close_log_window` - Close a log window
  - `focus_window` / `focus_window_by_index` - Focus windows
  - `get_all_windows` / `get_window_info` - Query window state
  - `set_window_index` - Change window keyboard shortcut index
  - `get_window_for_source` - Find window for a source
- Implemented `WindowManagerState` for tracking:
  - Window labels to source IDs mapping
  - Window indices (1-9) to window labels mapping
  - Automatic index assignment

### TypeScript Frontend - Window API
- Created `windowApi.ts` adapter with type-safe bindings:
  - All window commands mapped to TypeScript functions
  - Window detection utilities (`isMainWindow`, `isLogWindow`, `getSourceIdFromWindow`)
  - Event subscriptions for window lifecycle events

### Pinia Store - Window State
- Created `windowStore.ts` for reactive window state:
  - Tracks all open windows with `Map<string, LogWindow>`
  - Computed properties: `allWindows`, `focusedWindow`, `nextAvailableIndex`
  - Actions: `openLogWindow`, `closeWindow`, `focusWindow`, `switchToWindow`
  - Event listener setup for window lifecycle events

### Window Detection & Routing

- Main window renders `App.vue` (dashboard)
- Log windows render `LogWindow.vue` (dedicated viewer)

### Components

  - Clear, close, and dashboard navigation buttons
- Modified `App.vue` - Added window management:

  - Window index badges showing open windows
